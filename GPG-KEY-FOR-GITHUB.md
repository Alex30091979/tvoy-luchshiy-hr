# Как сделать коммиты Verified на GitHub (GPG)

Ваш ключ уже настроен для подписи коммитов (`user.signingkey 0482A9E0A4BBAD30`). Чтобы GitHub показывал **Verified**, в аккаунт нужно добавить **публичный** ключ.

## Шаг 1: Скопировать публичный ключ

В корне проекта лежит файл **`my-gpg-public-key.asc`** — это ваш публичный GPG-ключ.

Откройте этот файл в блокноте и скопируйте **весь** текст: от `-----BEGIN PGP PUBLIC KEY BLOCK-----` до `-----END PGP PUBLIC KEY BLOCK-----` (включительно).

## Шаг 2: Добавить ключ в GitHub

1. Откройте: https://github.com/settings/keys  
2. Нажмите **New GPG key**.  
3. В поле **Key** вставьте скопированный блок (BEGIN … END).  
4. Нажмите **Add GPG key**.

## Шаг 3: Проверка

Сделайте новый коммит и запушьте. На странице коммитов GitHub у него должен появиться бейдж **Verified**.

---

Если ключ уже добавлен, но коммиты всё равно Unverified:

- Убедитесь, что в GitHub добавлен именно **публичный** ключ (экспорт с `--export`, не `--export-secret-keys`).  
- Email в подписи коммита (`ra9680320325@gmail.com`) должен быть привязан к вашему аккаунту GitHub: https://github.com/settings/emails — при необходимости добавьте и подтвердите его.
